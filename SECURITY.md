# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in vue-why-did-you-render, please report it responsibly.

### How to Report

1. **Do NOT** open a public GitHub issue for security vulnerabilities.
2. Email the maintainer directly at the email associated with the [repository owner](https://github.com/iamdarshshah).
3. Alternatively, use GitHub's private vulnerability reporting feature if available.

### What to Include

Please provide the following information in your report:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (optional but appreciated)

### Response Timeline

- **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
- **Initial Assessment**: We will provide an initial assessment within 7 days.
- **Resolution**: We aim to resolve critical vulnerabilities within 30 days.

### After Reporting

- We will keep you informed about the progress of fixing the vulnerability.
- Once the issue is resolved, we will publicly acknowledge your contribution (unless you prefer to remain anonymous).
- We may ask for additional information or clarification during our investigation.

## Security Best Practices

When using vue-why-did-you-render:

1. **Development Only**: This library is intended for development and debugging purposes. Do not include it in production builds.
2. **Keep Updated**: Always use the latest version to benefit from security patches.
3. **Dependencies**: Regularly audit your dependencies using `npm audit` or `pnpm audit`.

## Disclosure Policy

We follow a coordinated disclosure policy:

1. Security issues are fixed in a private branch.
2. A new release is prepared with the fix.
3. The vulnerability is disclosed after users have had reasonable time to update.

Thank you for helping keep vue-why-did-you-render and its users safe!
